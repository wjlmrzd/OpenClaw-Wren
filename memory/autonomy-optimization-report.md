## 🔧 一、新增 Agent（自愈与协调）

### 1. 🚑 故障自愈员 (Auto-Healer)

**任务 ID**: `ccb233d7-0977-4d57-aba7-7564a67041d8`  
**执行频率**: 每 30 分钟  
**超时限制**: 300 秒

**职责**:
- 扫描最近 1 小时内失败的 cron 任务
- 分类错误类型（timeout / model_error / runtime_error）
- 自动执行修复：
  - `timeout` → 延迟重试（指数退避）
  - `model_error` → 自动切换到 fallback 模型（glm-5）
  - `runtime_error` → 尝试重启 gateway 或相关服务
- 连续失败 ≥3 次 → 发送 Telegram 高优先级告警
- 写入 `memory/incident-log.md`

**核心脚本**: `scripts/auto-healer.ps1`

**告警升级策略**:
```
1 次失败 → 自动修复，不通知
2 次失败 → 自动修复 + 记录
3 次失败 → Telegram 告警
5 次失败 → 禁用任务 + 紧急告警
```

### 2. 📡 事件协调员 (Event Hub)

**任务 ID**: `3a1df011-613d-4528-a274-530cfd84f4fb`  
**执行频率**: 每 5 分钟  
**超时限制**: 180 秒

**职责**:
- 收集所有 agent 输出的事件日志（统一写入 `memory/events.log`）
- 识别关键系统状态：
  - 内存 > 90% → 触发日志清理任务
  - 磁盘 > 85% → 强制日志清理
  - API 使用 > 80% → 降低高频 agent 执行频率
  - Gateway 异常 → 通知 auto_healer
- 记录处理动作到 `memory/event-hub-state.json`

**核心脚本**: `scripts/event-hub.ps1`

**联动规则**:
| 触发条件 | 联动动作 | 优先级 |
|---------|---------|--------|
| 内存 > 95% | 重启 Gateway | 🔴 紧急 |
| 内存 > 90% | 触发日志清理 | 🟡 警告 |
| 磁盘 > 95% | 强制清理 + 告警 | 🔴 紧急 |
| 磁盘 > 85% | 触发日志清理 | 🟡 警告 |
| API > 90% | 延迟非关键任务 | 🟡 警告 |
| Gateway 异常 | 通知 auto_healer | 🔴 紧急 |

## 🔧 三、核心脚本

### scripts/auto-healer.ps1

**功能**: 自动检测并修复系统故障

**主要能力**:
- Cron 任务错误扫描
- 系统资源监控（内存/磁盘）
- Gateway 状态检查
- 事件日志分析
- 自动修复执行

**使用方式**:
```powershell
# 完整扫描
.\auto-healer.ps1

# 干运行（不执行修复）
.\auto-healer.ps1 -DryRun

# 详细输出
.\auto-healer.ps1 -Verbose
```

### scripts/event-hub.ps1

**功能**: 事件协调中心，收集事件并触发联动

**主要能力**:
- 事件日志读取与分析
- 系统状态检查（内存/磁盘/Gateway/Cron）
- 联动动作触发
- 状态持久化

**使用方式**:
```powershell
# 完整检查 + 触发联动
.\event-hub.ps1 -TriggerActions

# 仅分析
.\event-hub.ps1 -Analyze

# 生成报告
.\event-hub.ps1 -Report
```

### scripts/event-logger.ps1

**功能**: 标准化事件日志记录

**使用方式**:
```powershell
.\event-logger.ps1 `
  -EventType "memory_critical" `
  -Message "内存使用率 95%" `
  -Level "error" `
  -Source "event_hub"
```

### scripts/scheduler-optimizer.ps1

**功能**: 调度优化分析

**使用方式**:
```powershell
# 分析当前调度
.\scheduler-optimizer.ps1 -Analyze

.\scheduler-optimizer.ps1 -Optimize

# 输出详细报告
.\scheduler-optimizer.ps1 -Report
```

## 🔄 五、闭环机制

### 事件日志格式

所有 agent 必须输出标准事件日志（JSON 格式）到 `memory/events.log`:

```json
{
  "ts": "2026-03-24T10:00:00.000Z",
  "tsUnix": 1774316400,
  "type": "task_completed",
  "level": "success",
  "source": "daily_report",
  "message": "每日早报生成成功",
  "metadata": {
    "duration": 120,
    "modules": 4
  }
}
```

### 事件级别定义

| 级别 | 说明 | Telegram 通知 |
|-----|------|--------------|
| `success` | 任务成功 | ❌ |
| `info` | 信息性事件 | ❌ |
| `warning` | 需要注意 | ✅ (摘要，静默时段延迟) |
| `error` | 错误事件 | ✅ (立即) |

### 联动流程

```
┌─────────────┐
│ Event Hub   │ 每 5 分钟检查
│ (事件协调)  │
└──────┬──────┘
       │
       ├─ 内存>90% ──────→ 触发 日志清理员
       │
       ├─ 磁盘>85% ──────→ 触发 日志清理员
       │
       ├─ Gateway 异常 ───→ 通知 Auto-Healer
       │
       └─ Cron 错误≥3 ────→ 通知 Auto-Healer
       
┌─────────────┐
│ Auto-Healer │ 每 30 分钟检查
│ (故障自愈)  │
└──────┬──────┘
       │
       ├─ 超时任务 ───────→ 增加 timeoutSeconds
       │
       ├─ 模型错误 ───────→ 验证配置 + 重置计数
       │
       └─ 连续失败≥3 ─────→ Telegram 告警
```

### 故障排查

**常见问题**:

1. **任务持续失败**
   - 检查 `memory/events.log` 中的错误详情
   - 运行 `auto-healer.ps1 -Verbose` 查看详细诊断
   - 手动执行任务：`openclaw cron run <jobId>`

2. **内存持续过高**
   - 检查 Gateway 进程内存使用
   - 考虑重启 Gateway：`openclaw gateway restart`
   - 增加日志清理频率

3. **事件日志不更新**
   - 检查 `event-logger.ps1` 脚本权限
   - 验证 `memory/` 目录写入权限
   - 检查 PowerShell 执行策略

### 配置调整

**调整任务频率**:
```powershell

notepad D:\OpenClaw\.openclaw\cron\jobs.json

# 重新加载配置
openclaw gateway restart
```

**调整资源阈值**:
编辑 `scripts/event-hub.ps1` 中的 `$thresholds` 变量。
