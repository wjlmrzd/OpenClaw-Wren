## 🧩 一、缺失的关键 Agent（按优先级）

### 1️⃣ 🚑 故障自愈 Agent（Auto-Healer）- ⭐⭐⭐ 必须
**执行频率：** 每 30 分钟

**职责：**
- 扫描最近失败的 cron 任务
- 分类错误类型：超时 / 模型错误 / 脚本异常
- 自动执行修复策略：
  - retry（带指数退避）
  - 替换 fallback 模型
  - 重启 Gateway
- 连续失败 >3 次 → 升级告警（Telegram）

**核心价值：** 系统闭环的核心，没有它只是"监控系统"，不是"自治系统"

---

### 2️⃣ 🧠 调度优化 Agent（Scheduler Optimizer）- ⭐⭐ 关键
**执行频率：** 每 6 小时

**职责：**
- 分析 cron 执行时间分布
- 自动调整任务时间（错峰）
- 避免：大任务重叠、API 峰值
- 输出优化后的 jobs.json

**当前撞车点：**
- 08:00 → 早报 + 网站监控
- 09:00 → 运营总监
- 23:00 → 备份（超重任务）

---

### 3️⃣ 📡 事件协调 Agent（Event Hub）- ⭐⭐ 关键
**执行频率：** 持续监听（或每 5 分钟轮询）

**职责：**
- 接收各 Agent 的"事件"
- 做决策联动：
  - 内存高 → 触发日志清理员
  - API 配额高 → 降低某些 Agent 频率
  - Gateway 挂 → 通知 Auto-Healer

**核心价值：** 从"脚本集合"→"多 Agent 协同系统"

---

### 4️⃣ 🧪 回归测试 Agent（Test Runner）- ⭐ 高级
**执行频率：** 代码变更后触发

**职责：**
- 自动生成测试用例
- 跑基础 smoke test
- 失败 → 阻止部署/提醒

## ⚠️ 二、现有 Agent 关键问题

### ❗1. 模型配置错误（最大坑）
**错误信息：** `Unknown model: anthropic/qwen3.5-plus`

**根本原因：** OpenClaw provider routing 配错了

**修复方案：**
```json
"model": "qwen/qwen3.5-plus"
// 或
"provider": "aliyun"
```

**原则：** 不要混用 `anthropic` ❌ `qwen` ❌

---

### ❗2. 超时任务（备份 / 日志 / 早报）

**根本原因：**
- IO 太重（git / 压缩）
- 单线程执行
- 没分页/分批

**优化方案：**

**备份管理员：**
- 分阶段：git add → commit → push（单独超时）
- 加：`--depth=1` 忽略大文件

**日志清理员：**
- 不要一次压缩全部
- 改为：每次最多处理 N 个文件（如 20 个）

**每日早报：**
- 拆任务：天气 / 新闻 / 邮件
- 并行 or 缓存

---

### ❗3. 静默策略升级

**当前：** 时间段控制（22:00-06:00）

**升级为：** "情境感知静默"
- 睡眠 + 非紧急 → 静默
- 工作时间 + 高频 → 限流
- 周末 → 降频
