### 系统概述
基于 **ClawTeam** 的 4 人代码评审团队，使用顺序工作流：

```
设计师 → 校核员 → 审核员 → 总工
```

每个角色由不同的 AI 模型担任，在独立的 tmux window 和 git worktree 中工作。

| 角色 | 模型 | 职责 |
|------|------|------|
| **设计师** | `qwen3-coder-plus` | 代码架构设计、初版实现 |
| **校核员** | `qwen3.5-plus` | 代码质量检查、风格审查 |
| **审核员** | `minimax-m2.5` ⭐ | 安全性审查、最佳实践 |
| **总工** | `glm-5` | 最终审批、合并决策 |

### 工作流程
```
用户请求
    ↓
[ClawTeam 团队启动]
    ↓
设计师 (独立 tmux + git worktree) → 生成代码
    ↓
校核员 (独立 tmux + git worktree) → 质量检查
    ↓
审核员 (独立 tmux + git worktree) → 安全审查
    ↓
总工 (独立 tmux + git worktree) → 最终审批
    ↓
合并到 D:\OpenClaw\plugins\ + Telegram 报告
```

### 文件位置

### 使用方法
**推荐方式**：在 Telegram 中说"生成 XX 插件"或"创建 XX 代码"

**手动方式**：
```powershell
cd D:\OpenClaw\.openclaw\workspace\scripts
.\launch-code-review.ps1 -Request "生成一个天气查询插件"
```

### ClawTeam 优势（相比独立 cron）
1. ✅ **内置依赖管理** - 任务自动按顺序执行（`--blocked-by`）
2. ✅ **Git worktree 隔离** - 每个 agent 独立分支，不会冲突
3. ✅ **内置消息系统** - agent 之间通过 inbox 通信
4. ✅ **看板监控** - `clawteam board show/live/attach`
5. ✅ **Web 面板** - 可视化监控所有 agent 进度
6. ✅ **成本追踪** - 自动统计各模型 token 使用
7. ✅ **自动清理** - 完成后一键清理团队资源

### 监控命令
```powershell
# 看板视图
python -m clawteam board show {team-name}

# 实时监控
python -m clawteam board live {team-name}

# Tmux 并排视图（所有 agent 同屏）
python -m clawteam board attach {team-name}

# Web 面板
python -m clawteam board serve --port 8080
```

### 问题
任务审计发现 5 个 Cron 任务处于错误状态：
1. **项目顾问** - 模型错配：`Unknown model: anthropic/qwen3.5-plus`
2. **Workspace 备份** - 超时 (120 秒)

### 修复措施
1. ✅ 更新 openclaw.json 添加所有 7 个 Coding Plan 模型
2. ✅ 配置完整的模型别名映射
3. ✅ 重启 Gateway 应用配置
4. ✅ 调整超时任务的 timeoutSeconds：
   - Workspace 备份：600→900 秒
   - 日志清理员：600→900 秒
   - 日历检查：300→450 秒
   - 每日早报：450→600 秒

### 状态
- 配置已修复，Gateway 已重启
- 错误状态将在下次成功执行后自动清除
- 已发送修复通知到 Telegram

### 教训
- 配置模型时必须确保所有 Cron 任务使用的模型都已定义
- 定期检查 Cron 任务的 lastError 字段
- 超时时间应根据任务复杂度合理设置
