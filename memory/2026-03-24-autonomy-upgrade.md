**问题：** `Unknown model: anthropic/qwen3.5-plus`

**根因：** Cron 任务中模型名称格式不统一，短别名被错误路由到 anthropic provider

**修复：**
- 统一所有 16 个 Cron 任务的 model 字段为完整路径格式
- `qwen3.5-plus` → `dashscope-coding-plan/qwen3.5-plus`
- `glm-5` → `dashscope-coding-plan/glm-5`
- `qwen3-coder-plus` → `dashscope-coding-plan/qwen3-coder-plus`
- `qwen3-coder-next` → `dashscope-coding-plan/qwen3-coder-next`

**状态：** ✅ 已完成，Gateway 已重启

### 2. Auto-Healer 故障自愈员（新增）

**执行频率：** 每 30 分钟  
**任务 ID：** `ccb233d7-0977-4d57-aba7-7564a67041d8`

**核心能力：**
- 🔍 扫描最近失败的 cron 任务
- 🏷️ 分类错误类型：超时 / 模型错误 / 脚本异常 / Gateway 错误
- 🔧 自动执行修复策略：
  - 超时 → 增加 timeoutSeconds 50%
  - 模型错误 → 确认配置已修复，重置计数

  - Gateway 错误 → 重启 Gateway
- 🚨 连续失败≥3 次 → 升级告警到 Telegram

**配套脚本：** `scripts/auto-healer.ps1`
- 状态追踪：`memory/auto-healer-state.json`
- 失败计数持久化
- 修复计划生成

**状态：** ✅ 已创建并启用

## 📋 待实施项目

### ⭐⭐ 第二步：关键质变

#### 1. Event Hub 事件协调员（待创建）
**执行频率：** 每 5 分钟轮询

**职责：**
- 接收各 Agent 的"事件"
- 决策联动：
  - 内存高 → 触发日志清理员
  - API 配额高 → 降低某些 Agent 频率
  - Gateway 挂 → 通知 Auto-Healer

**实现思路：**
- 创建共享事件队列：`memory/event-queue.json`
- 各 Agent 发布事件到队列
- Event Hub 消费并触发联动

## 💡 关键教训

1. **模型名称必须统一使用完整路径**，避免 provider 路由错误
2. **自愈机制是自治系统的核心**，没有它只是监控系统
3. **超时任务需要分批/并行处理**，避免 IO 密集型任务阻塞
4. **事件驱动架构** 是 Agent 协同的关键
