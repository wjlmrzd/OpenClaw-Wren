## 🔄 新增能力

### 1. 故障自愈 (Auto-Healer)

**能力：**
- 🔍 自动扫描失败任务
- 🔧 自动修复模型错误和超时问题
- 📢 连续失败≥3 次发送告警
- 🛡️ 修改前自动备份配置

**修复策略：**
```
第 1 次失败 → 自动修复，不通知
第 2 次失败 → 自动修复 + 记录
第 3 次失败 → Telegram 告警
第 5 次失败 → 禁用任务 + 紧急告警
```

### 2. 事件联动 (Event Hub)

**能力：**
- 📊 每 5 分钟检查系统状态
- 🔗 触发 Agent 联动（内存高→日志清理，任务失败→Auto-Healer）
- 📝 记录所有事件到 event-log.md
- 🎯 智能通知（仅警告/紧急事件）

**联动规则：**
```
MEM_HIGH (85%)     → 触发 🧹 日志清理员
MEM_CRITICAL (95%) → 触发 🚑 故障自愈员 (重启 Gateway)
TASK_REPEATED_FAIL → 通知 🚑 故障自愈员
GW_DOWN            → 尝试重启 + 告警
API_QUOTA_HIGH     → 延迟非关键任务
```
