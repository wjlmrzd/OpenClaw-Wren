# Telegram 控制指令文档

## 📖 概述

Telegram Commander 提供通过 Telegram 对 OpenClaw 系统进行实时控制、调试与干预的能力。

## 🎯 支持的指令

### 📊 系统状态类

| 指令 | 说明 | 权限 |
|------|------|------|
| `/status` | 查看系统状态（Gateway、任务、资源） | 白名单 |

**输出内容：**
- Gateway 运行状态
- 当前运行任务列表
- 最近失败任务
- CPU / 内存 / 磁盘使用率

---

### 🔁 任务控制类

| 指令 | 说明 | 权限 | 二次确认 |
|------|------|------|----------|
| `/jobs` | 列出所有 cron 任务 + 状态 | 白名单 | ❌ |
| `/retry [任务名]` | 立即重试失败任务 | 白名单 | ❌ |
| `/run [任务名]` | 手动执行任务 | 白名单 | ❌ |
| `/pause [任务名]` | 暂停某个任务 | 白名单 | ❌ |
| `/resume [任务名]` | 恢复任务 | 白名单 | ❌ |

**示例：**
```
/retry 每日早报
/run 健康监控员
/pause 网站监控员
```

**模糊匹配：** 任务名支持模糊匹配，如 `/retry 早报` 可匹配 `📰 每日早报`

---

### 🚨 故障处理类

| 指令 | 说明 | 权限 | 二次确认 |
|------|------|------|----------|
| `/fix` | 触发 auto_healer 立即执行 | 白名单 | ❌ |
| `/restart` | 重启 Gateway | 白名单 | ✅ |
| `/clean` | 触发日志清理 | 白名单 | ❌ |

---

### ⚙️ 系统调节类

| 指令 | 说明 | 权限 | 二次确认 |
|------|------|------|----------|
| `/safe-mode` | 启用安全模式（降低负载） | 白名单 | ✅ |
| `/normal-mode` | 恢复正常调度 | 白名单 | ❌ |

**安全模式效果：**
- 降低所有非关键任务执行频率 50%
- 暂停高耗资源任务
- 仅保留核心监控任务

---

### 📡 调试类

| 指令 | 说明 | 权限 |
|------|------|------|
| `/logs [任务名]` | 查看最近执行日志 | 白名单 |
| `/events` | 查看最近系统事件 | 白名单 |

---

### 📖 帮助

| 指令 | 说明 |
|------|------|
| `/help` | 显示指令帮助 |

---

## 🔐 安全与权限

### 白名单机制
- 仅 `scripts/telegram-whitelist.json` 中配置的用户可执行指令
- 默认允许用户：`8542040756`

### 二次确认
以下高危操作需要二次确认（5 分钟内再次发送相同指令）：
- `/restart` - 重启 Gateway
- `/safe-mode` - 启用安全模式

### 操作日志
所有操作记录到：`memory/telegram-ops-log.md`

日志格式：
```markdown
## 2026-03-24 10:00:00

- **用户**: 8542040756
- **指令**: /restart
- **结果**: SUCCESS
- **详情**: Gateway 重启中...
```

---

## 🔁 联动机制

Telegram Commander 与以下 Agent 联动：

| Agent | 联动方式 |
|-------|----------|
| 🚑 故障自愈员 | `/fix` 触发自动修复 |
| 📡 事件协调员 | `/events` 获取系统事件 |
| 🧠 调度优化员 | `/safe-mode` 调整调度 |
| ⚖️ 资源守护者 | `/status` 返回资源数据 |

---

## 📝 使用示例

### 查看系统状态
```
/status
```

### 重试失败任务
```
/retry 每日早报
```

### 暂停任务
```
/pause 网站监控员
```

### 重启 Gateway（需二次确认）
```
/restart
# 系统回复确认提示
/restart
# 确认执行
```

### 查看帮助
```
/help
```
