## 常见问题

| 问题 | 检查 | 解决 |
|-----|------|------|
| Cron 任务不执行 | 错误恢复官报告 | 查看 lastError 字段 |
| Gateway 崩溃 | Gateway 健康监控 | 检查内存/进程状态 |
| 配置丢失 | 配置审计师备份 | 从 `memory/config-backups/` 恢复 |
| 磁盘空间不足 | 日志清理员报告 | 手动清理或触发清理任务 |
| API 请求失败 | 资源守护者报告 | 检查配额和网络 |

---

## 紧急恢复流程

### 1. 检查 Gateway 状态

```powershell
openclaw gateway status
```

### 2. 查看错误日志

```powershell
openclaw logs --follow
```

### 3. 恢复配置（如需要）

```powershell
Copy-Item memory\config-backups\openclaw-*.json openclaw.json -Force
```

### 4. 重启 Gateway

```powershell
openclaw gateway restart
```

---

## 故障自愈流程

```
故障检测 (每 30 分钟)
         ↓
   错误恢复官扫描
         ↓
   识别失败任务
         ↓
   自动重试 (3 次)
         ↓
   模型切换 (如需要)
         ↓
   Gateway 重启 (如需要)
         ↓
   发送报告/告警
```

---

## 告警级别

| 级别 | 触发条件 | 通知策略 |
|------|---------|----------|
| 🟢 信息 | 正常运行 | 工作时间发送 |
| 🟡 警告 | 可自愈问题 | 工作时间发送 |
| 🔴 紧急 | 需关注问题 | 任何时段发送 |
| 🔴🔴 危急 | 系统故障 | 立即发送 + 冻结保护 |

---

## 静默时段策略

**静默时段**: 22:00-06:00

| 级别 | 工作时间 | 傍晚 | 深夜 |
|------|---------|------|------|
| 🟢 信息 | ✅ | ❌ | ❌ |
| 🟡 警告 | ✅ | ✅ | ❌ (累积) |
| 🔴 紧急 | ✅ | ✅ | ✅ |
| 🔴🔴 危急 | ✅ | ✅ | ✅ |

**紧急事件例外** (任何时段立即通知):
- Gateway 无法启动/重启失败
- 内存/磁盘使用率 > 95%
- 安全事件（未授权访问、凭证泄露）
- 关键任务连续失败 ≥ 5 次
