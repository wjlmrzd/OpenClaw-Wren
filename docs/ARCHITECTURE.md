## 🎯 任务分发流程

### 用户请求 → 主会话解析

```
用户："帮我分析一下这个项目的代码结构"
         ↓
   主会话 (qwen3.5-plus)
         ↓
   识别为"代码开发类"
         ↓
   创建子代理 (qwen3-coder-plus)
         ↓
   实时返回分析结果
```

### 周期性任务 → Cron 执行

```
定时触发 (如每天 8:00)
         ↓
   Cron 调度器
         ↓
   创建 isolated 会话
         ↓
   执行指定任务
         ↓
   结果推送给用户
```

### 综合报告 → 任务协调员整合

```
每天 21:00
         ↓
   任务协调员启动
         ↓
   收集 17 个 Cron 任务结果
         ↓
   识别跨任务事项
         ↓
   生成综合日报
         ↓
   发送给用户
```

## 📞 故障排查

### 常见问题

| 问题 | 检查 | 解决 |
|-----|------|------|
| Cron 任务不执行 | 错误恢复官报告 | 查看 lastError 字段 |
| Gateway 崩溃 | Gateway 健康监控 | 检查内存/进程状态 |
| 配置丢失 | 配置审计师备份 | 从 memory/config-backups/ 恢复 |
| 磁盘空间不足 | 日志清理员报告 | 手动清理或触发清理任务 |
| API 请求失败 | 资源守护者报告 | 检查配额和网络 |

### 紧急恢复流程
```powershell
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 查看错误日志
openclaw logs --follow

# 3. 恢复配置（如需要）
Copy-Item memory\config-backups\openclaw-*.json openclaw.json -Force

# 4. 重启 Gateway
openclaw gateway restart
```
