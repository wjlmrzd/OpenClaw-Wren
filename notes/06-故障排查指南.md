# 故障排查指南

## 🔍 常见问题速查

### Gateway 无法启动

**症状**:
- `openclaw gateway start` 失败
- 端口被占用
- 配置错误

**排查步骤**:
```powershell
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 查看详细错误
openclaw logs --follow

# 3. 检查端口占用
netstat -ano | findstr :18789

# 4. 验证配置
openclaw config.get

# 5. 尝试修复
.\scripts\openclaw-auto-fix.bat

# 6. 重启 Gateway
openclaw gateway restart
```

**常见原因**:
- ❌ 配置文件语法错误
- ❌ 端口被占用
- ❌ 凭证文件缺失
- ❌ Node.js 版本不兼容

---

### Cron 任务不执行

**症状**:
- 任务未按时触发
- 任务执行失败
- 无输出报告

**排查步骤**:
```powershell
# 1. 查看 Cron 任务列表
openclaw cron list

# 2. 查看任务运行历史
openclaw cron runs --jobId=<任务 ID>

# 3. 检查 lastError 字段
openclaw cron list --includeDisabled

# 4. 手动触发任务
openclaw cron run --jobId=<任务 ID>

# 5. 查看错误日志
type memory\events.log

# 6. 运行回归测试
.\scripts\test-runner.ps1
```

**常见原因**:
- ❌ 模型配置错误
- ❌ Cron 表达式无效
- ❌ 会话创建失败
- ❌ API 配额耗尽

---

### 消息发送失败

**症状**:
- Telegram 无响应
- 消息发送超时
- 白名单拒绝

**排查步骤**:
```powershell
# 1. 检查白名单配置
type credentials\telegram-allowFrom.json

# 2. 验证 Bot Token
# 检查 .env 文件 TELEGRAM_BOT_TOKEN

# 3. 测试 Gateway 连接
curl http://127.0.0.1:18789/status

# 4. 查看消息日志
type memory\events.log | Select-String "message"

# 5. 检查网络连接
ping api.telegram.org
```

**常见原因**:
- ❌ Bot Token 无效
- ❌ 用户不在白名单
- ❌ 网络问题
- ❌ Gateway 未运行

---

### 模型调用失败

**症状**:
- `Error: Unknown model: xxx`
- API 返回错误
- 响应超时

**排查步骤**:
```powershell
# 1. 检查模型配置
openclaw config.get | ConvertFrom-Json | Select-Object -ExpandProperty agents

# 2. 验证模型别名
# 参考：notes/03-模型配置说明.md

# 3. 测试 API 连接
# 检查 Coding Plan API 状态

# 4. 查看错误详情
type memory\events.log | Select-String "model"

# 5. 切换备用模型
# 修改 openclaw.json 中的 model.primary
```

**常见原因**:
- ❌ 模型名称拼写错误
- ❌ API Base URL 错误
- ❌ 凭证过期
- ❌ 网络问题

---

### 磁盘空间不足

**症状**:
- 写入失败
- 日志无法保存
- 系统变慢

**排查步骤**:
```powershell
# 1. 检查磁盘空间
Get-PSDrive C | Select-Object Used,Free

# 2. 查找大文件
.\scripts\find-large-files.ps1

# 3. 清理临时文件
.\scripts\clean-temp.ps1

# 4. 清理缓存
.\scripts\check-cache-wps.ps1
.\scripts\check-other-caches.ps1

# 5. 清理日志
.\scripts\log-rotate.ps1

# 6. 查看清理报告
type memory\event-log.md
```

**常见原因**:
- ❌ 日志文件积累
- ❌ 缓存文件过多
- ❌ 备份文件未清理
- ❌ 临时文件堆积

---

### 内存使用过高

**症状**:
- 系统响应慢
- Gateway 崩溃
- 任务执行失败

**排查步骤**:
```powershell
# 1. 检查内存使用
Get-Process openclaw | Select-Object CPU,WorkingSet

# 2. 查看资源报告
# 资源守护者每 4 小时发送

# 3. 重启 Gateway
openclaw gateway restart

# 4. 清理缓存
.\scripts\check-cache-wps.ps1

# 5. 暂停低优先级任务
# 修改 cron 任务 enabled 字段
```

**常见原因**:
- ❌ 内存泄漏
- ❌ 并发任务过多
- ❌ 缓存未清理
- ❌ 模型上下文过大

---

### 配置丢失或损坏

**症状**:
- 配置无法加载
- 设置丢失
- 系统行为异常

**排查步骤**:
```powershell
# 1. 检查配置文件
type openclaw.json

# 2. 验证 JSON 语法
# 使用在线 JSON 验证工具

# 3. 从备份恢复
Copy-Item memory\config-backups\openclaw-*.json openclaw.json -Force

# 4. 重启 Gateway
openclaw gateway restart

# 5. 验证配置
openclaw config.get
```

**预防措施**:
- ✅ 配置审计师每 4 小时备份
- ✅ Git 每天自动提交
- ✅ 修改前先备份

## 🚨 紧急情况处理

### Gateway 完全无法启动

```powershell
# 1. 强制停止所有进程
taskkill /F /IM node.exe

# 2. 清理端口
netstat -ano | findstr :18789
# 记录 PID，然后 taskkill /F /PID <pid>

# 3. 恢复配置
Copy-Item memory\config-backups\openclaw-*.json openclaw.json -Force

# 4. 重新启动
openclaw gateway start

# 5. 如果仍失败，查看完整日志
openclaw logs --follow
```

### 系统进入安全模式

**触发条件**:
- 内存/磁盘 > 95%
- Gateway 重启失败 ≥ 3 次
- 连续错误 ≥ 10 次

**恢复步骤**:
```powershell
# 1. 查看当前模式
type memory\system-mode-state.json

# 2. 解决根本问题
# - 清理磁盘空间
# - 修复 Gateway 配置
# - 检查错误日志

# 3. 手动切换回正常模式
.\scripts\system-mode-tools.ps1 -Mode normal

# 4. 重启 Gateway
openclaw gateway restart

# 5. 验证系统正常
openclaw gateway status
```
