# 安全机制文档

## 🛡️ 安全体系概览

OpenClaw 系统采用多层安全防护机制，确保数据和系统安全。

---

## 🔐 访问控制

### 1. Telegram 白名单模式

**配置位置**: `credentials/telegram-allowFrom.json`

```json
{
  "allowFrom": [8542040756]
}
```

**规则**:
- ✅ 仅允许授权用户 ID 访问
- ✅ 群组策略：白名单模式
- ✅ 拒绝所有未授权请求

### 2. Gateway 绑定

**配置**:
- **绑定地址**: 127.0.0.1 (仅本地)
- **端口**: 18789
- **模式**: Local

**效果**:
- ✅ 外部网络无法直接访问
- ✅ 仅本机可连接 Gateway
- ✅ 如需远程访问需配置 Tailscale

## 🔒 配置保护

### 1. 配置审计师

**频率**: 每 4 小时

**职责**:
- 监控配置文件变更
- 自动备份配置到 `memory/config-backups/`
- 检测未授权修改
- 发送审计报告

### 2. 配置备份

**备份位置**: `memory/config-backups/`

**备份内容**:
- openclaw.json
- cron/jobs.json
- credentials/*
- agents/*/models.json

**保留策略**:
- 保留最近 7 天配置
- 每天自动清理旧备份

### 3. 敏感文件保护

**.gitignore 排除**:
```
.env
credentials/
*.log
memory/config-backups/
```

**原则**:
- ✅ 绝不提交敏感文件到 Git
- ✅ 凭证文件本地存储
- ✅ 日志文件定期清理

## 🚨 错误恢复

### 1. 故障自愈员

**频率**: 每 30 分钟

**自动修复能力**:
- Gateway 崩溃 → 自动重启
- 任务失败 → 自动重试 (3 次)
- 模型错误 → 切换备用模型
- 配置错误 → 从备份恢复

### 2. 重试机制

```
任务失败
    ↓
等待 1 分钟 → 重试 (第 1 次)
    ↓
等待 5 分钟 → 重试 (第 2 次)
    ↓
等待 15 分钟 → 重试 (第 3 次)
    ↓
发送告警通知
```

### 3. 事件升级

| 连续错误次数 | 动作 |
|------------|------|
| 3 次 | 发送告警通知 |
| 5 次 | 暂停任务 |
| 10 次 | 冻结系统，等待人工 |

## 📊 资源监控

### 1. 资源守护者

**频率**: 每 4 小时

**监控指标**:
- CPU 使用率
- 内存使用率
- 磁盘空间
- API 请求频率

### 2. 预警阈值

| 指标 | 警告 | 严重 | 危急 |
|------|------|------|------|
| 内存 | 85% | 90% | 95% |
| 磁盘 | 85% | 90% | 95% |
| API 请求 | 80% | 90% | 95% |

### 3. 自动保护

- **85%**: 发送警告，准备清理
- **90%**: 清理缓存，暂停低优先级任务
- **95%**: 进入安全模式，仅保留核心任务

## 📝 日志管理

### 1. 统一日志系统

**日志文件**:
- `memory/events.log` - 结构化日志
- `memory/event-log.md` - Markdown 日志
- `memory/incident-log.md` - 事件升级记录

**格式**:
```
[YYYY-MM-DD HH:mm:ss] [SEVERITY] [SOURCE] message | key=value
```

### 2. 日志级别

| 级别 | 说明 | 示例 |
|------|------|------|
| INFO | 信息 | 任务执行成功 |
| WARN | 警告 | 资源使用率高 |
| ERROR | 错误 | 任务执行失败 |
| CRITICAL | 危急 | 系统崩溃 |

### 3. 日志清理

**频率**: 每天 03:00

**策略**:
- 保留最近 7 天详细日志
- 压缩 7-30 天日志
- 删除 30 天以上日志

## 🚑 灾难恢复

### 1. 恢复流程

```
系统故障
    ↓
故障自愈员尝试自动修复 (3 次)
    ↓
失败 → 发送告警
    ↓
人工介入
    ↓
从备份恢复配置
    ↓
重启 Gateway
    ↓
验证系统正常
```

### 2. 恢复命令

```powershell
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 查看错误日志
openclaw logs --follow

# 3. 恢复配置
Copy-Item memory\config-backups\openclaw-*.json openclaw.json -Force

# 4. 重启 Gateway
openclaw gateway restart
```
