## 🎯 任务分发流程

### 用户请求 → 主会话解析

```
用户："帮我分析一下这个项目的代码结构"
         ↓
   主会话 (qwen3.5-plus)
         ↓
   识别为"代码开发类"
         ↓
   创建子代理 (qwen3-coder-plus)
         ↓
   实时返回分析结果
```

### 周期性任务 → Cron 执行

```
定时触发 (如每天 8:00)
         ↓
   Cron 调度器
         ↓
   创建 isolated 会话
         ↓
   执行指定任务
         ↓
   结果推送给用户
```

### 综合报告 → 任务协调员整合

```
每天 21:00
         ↓
   任务协调员启动
         ↓
   收集所有 Cron 任务结果
         ↓
   识别跨任务事项
         ↓
   生成综合日报
         ↓
   发送给用户
```

## 🔄 数据流向

1. **入站流量**: Telegram → Gateway → 主会话
2. **出站流量**: 主会话 → Gateway → Telegram
3. **定时任务**: Cron 调度器 → Isolated 会话 → Gateway → Telegram
4. **子代理**: 主会话 → Subagent 会话 → 主会话 → 用户

## ⚙️ 配置层次

```
openclaw.json (根配置)
    ├── channels.telegram    # Telegram 通道配置
    ├── agents.defaults      # Agent 默认配置

    │   └── models           # 可用模型列表

    └── delivery             # 消息投递配置
```
