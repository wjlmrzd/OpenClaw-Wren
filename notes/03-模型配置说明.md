# 模型配置说明

## 🎯 模型概览

系统配置了 **7 款** AI 模型，均通过阿里云百炼 Coding Plan 接入。

---

## 📋 模型列表

### 1. qwen3.5-plus ⭐ 主模型
- **完整名称**: dashscope-coding-plan/qwen3.5-plus
- **类型**: 通用旗舰模型
- **参数**: 3970 亿
- **特点**: 综合能力最强，适合复杂任务
- **用途**: 
  - 主会话对话
  - 综合报告生成
  - 任务协调
- **配置位置**: `agents.defaults.model.primary`

### 2. qwen3-coder-plus
- **完整名称**: dashscope-coding-plan/qwen3-coder-plus
- **类型**: 编程增强版
- **特点**: 代码理解和生成能力强
- **用途**:
  - 代码分析
  - 系统监控任务
  - 故障自愈
- **配置位置**: `agents.defaults.models.qwen3-coder-plus`

### 3. qwen3-coder-next
- **完整名称**: dashscope-coding-plan/qwen3-coder-next
- **类型**: 编程专用
- **特点**: 轻量快速，适合简单代码任务
- **用途**:
  - 代码生成
  - 备份任务
  - 日志清理
- **配置位置**: `agents.defaults.models.qwen3-coder-next`

### 4. glm-5
- **完整名称**: dashscope-coding-plan/glm-5
- **类型**: 智谱旗舰
- **特点**: 长文档处理能力强
- **用途**:
  - 安全审计
  - 每周总结
  - 灾难恢复
- **配置位置**: `agents.defaults.models.glm-5`

### 5. glm-4.7
- **完整名称**: dashscope-coding-plan/glm-4.7
- **类型**: 智谱 GLM 4.7 版本
- **特点**: 备用模型
- **用途**:
  - 模型故障切换
  - 备用任务
- **配置位置**: `agents.defaults.models.glm-4.7`

### 6. kimi-k2.5
- **完整名称**: dashscope-coding-plan/kimi-k2.5
- **类型**: 多模态和编程
- **特点**: 多模态处理能力强
- **用途**:
  - 邮件监控
  - 图像分析
- **配置位置**: `agents.defaults.models.kimi-k2.5`

### 7. minimax-m2.5
- **完整名称**: dashscope-coding-plan/minimax-m2.5
- **类型**: Agent 场景专用
- **特点**: 办公场景优化
- **用途**:
  - 日程管理
  - 办公自动化
- **配置位置**: `agents.defaults.models.minimax-m2.5`

## 🎯 模型选择策略

### 按任务类型选择

| 任务类型 | 推荐模型 | 理由 |
|---------|---------|------|
| 通用对话 | qwen3.5-plus | 综合能力最强 |
| 代码开发 | qwen3-coder-plus | 编程增强 |
| 简单代码 | qwen3-coder-next | 快速轻量 |
| 长文档分析 | glm-5 | 长文本优化 |
| 多模态 | kimi-k2.5 | 图像 + 文本 |
| 办公场景 | minimax-m2.5 | Agent 优化 |
| 备用 | glm-4.7 | 降级方案 |

### 按优先级选择

1. **高优先级任务**: qwen3.5-plus / qwen3-coder-plus
2. **中优先级任务**: glm-5 / kimi-k2.5
3. **低优先级任务**: qwen3-coder-next / minimax-m2.5

---

## 🔄 故障切换策略

```
主模型 (qwen3.5-plus) 失败
    ↓
备用模型 1 (glm-5)
    ↓
备用模型 2 (kimi-k2.5)
    ↓
发送告警通知
```

## ⚠️ 注意事项

### Coding Plan 限制
- **计费方式**: 固定月费 (非 Token 计费)
- **限制**: 每小时请求次数限制
- **监控**: 资源守护者每 2 小时检查

### 最佳实践
1. 根据任务类型选择合适的模型
2. 避免单一模型过度使用
3. 定期检查模型配置有效性
4. 保持备用模型可用
