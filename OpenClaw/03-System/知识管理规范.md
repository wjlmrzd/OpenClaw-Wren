# {{知识管理规范}}

## 概述

这是 OpenClaw 第二大脑的核心规范，定义通过文件系统方式管理 Obsidian Vault 的所有规则。所有笔记直接写入 Markdown 文件，不依赖任何 UI 或插件 API。

## 核心要点

- **文件系统优先** - 直接读写 .md 文件，UTF-8 编码
- **PARA 结构** - 00-Inbox/01-Knowledge/02-Projects/03-System/04-Issues
- **双链关联** - 使用 [[主题名称]] 建立关系
- **自动整理** - 每日扫描、修复、优化

## 详细说明

### 目录结构

```
knowledge/
├── 00-Inbox/        # 临时收集，待分类
├── 01-Knowledge/    # 通用知识、概念、理论
├── 02-Projects/     # 进行中项目、任务
├── 03-System/       # 系统设计、架构、规范
└── 04-Issues/       # 问题记录、解决方案
```

### 笔记格式

```markdown
# {{主题名称}}

## 概述
...

## 核心要点
- ...

## 详细说明
...

## 相关概念
- [[双链]]

---
tags: []
created: YYYY-MM-DD
source: openclaw
type: note
```

### 文件命名

- 文件名：`主题名称.md`
- 编码：UTF-8（无 BOM）
- 路径：遵循 PARA 分类

### 双链规则

- 格式：`[[主题名称]]`
- 自动检测目标文件是否存在
- 不存在时创建空壳笔记
