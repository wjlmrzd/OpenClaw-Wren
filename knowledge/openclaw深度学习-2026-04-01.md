## 1️⃣ commands.native / nativeSkills: "auto" 的具体行为

### 核心机制

`nativeSkills` 和 `commands.native` 控制是否在渠道（Discord/Telegram/Slack等）注册**原生命令菜单**。

**关键代码** (`dist/config/commands.js`):

```javascript
function resolveAutoDefault(providerId) {
  const id = normalizeChannelId(providerId);
  if (id === "discord" || id === "telegram") return true;   // ✅ 自动启用
  if (id === "slack") return false;                          // ❌ 需手动启用
  return false;
}
```

### "auto" 的含义

| 设置值 | 行为 |
|--------|------|
| `"auto"` 或 `undefined` | 使用渠道默认值 |
| `true` | 强制启用原生命令 |
| `false` | 强制禁用原生命令 |

### 渠道默认值矩阵

| 渠道 | 默认行为 | 原因 |
|------|----------|------|
| **Discord** | `true` ✅ | Slash Commands 是 Discord 核心交互方式 |
| **Telegram** | `true` ✅ | Bot Commands 菜单原生支持 |
| **Slack** | `false` ❌ | Slash Commands 需要显式配置 App Manifest |
| **其他** | `false` ❌ | 飞书、钉钉、企业微信等不支持原生菜单 |

### 配置层次

```json5
{
  // 全局设置
  commands: {
    native: "auto",       // 或 true/false
    nativeSkills: "auto"  // 或 true/false
  },
  
  // 渠道覆盖（优先级更高）
  channels: {
    discord: { commands: { native: true } },
    slack: { commands: { native: true } },   // 手动启用
    telegram: { commands: { native: false } } // 手动禁用
  }
}
```

### 实际效果

- **启用时**: 渠道注册 `/new`, `/reset`, `/status` 等命令到平台原生菜单
- **禁用时**: 用户只能通过文本输入触发命令（如 `new`, `/new`, `新对话`）

### Slack 特殊情况

Slack 需要额外配置：
1. 在 Slack App Manifest 中定义 Slash Commands
2. 或通过 `openclaw slack commands sync` 同步（如果支持）

## 2️⃣ models.mode: "merge" 的合并策略

### 核心机制

`models.mode` 控制 `models.providers` 配置如何写入 `models.json`。

**关键代码** (`dist/agents/models-config.js`):

```javascript
const DEFAULT_MODE = "merge";

// merge 模式：读取现有 models.json，浅合并
if (mode === "merge") {
  const existing = await readJson(targetPath);
  if (isRecord(existing) && isRecord(existing.providers)) {
    mergedProviders = { ...existingProviders, ...providers };
  }
}

// replace 模式：直接覆盖（不读取现有）
// 无特殊处理，直接用 providers
```

### 合 vs 替换

| 设置值 | 行为 | 适用场景 |
|--------|------|----------|
| `"merge"` (默认) | 浅合并 providers | 渐进式添加模型 |
| `"replace"` | 完全覆盖 providers | 重置模型配置 |

### 合逻辑详解

```javascript
// 浅合并 - 新 providers 覆盖同名键
mergedProviders = { ...existingProviders, ...providers };

// 示例：
existing = { "openai": {...}, "anthropic": {...} }
new     = { "anthropic": {新配置} }
result  = { "openai": {...}, "anthropic": {新配置} }  // anthropic 被替换
```

### Provider 内部的 Models 合

```javascript
function mergeProviderModels(implicit, explicit) {
  // 模型 ID 去重：explicit 模型优先
  const seen = new Set(explicitModels.map(getId));
  const mergedModels = [
    ...explicitModels,
    ...implicitModels.filter(m => !seen.has(m.id))
  ];
}
```

### 实际流程

1. Gateway 启动 → `ensureOpenClawModelsJson()`
2. 读取 `openclaw.json` → 提取 `models.providers`
3. 发现隐式 providers（如目录下的 `.openai-api-key` 文件）
4. 根据 `mode` 合或替换 → 写入 `~/.openclaw/models.json`
5. Agent 使用 `models.json` 作为模型 catalog

### 配置示例

```json5
{
  models: {
    mode: "merge",   // 或 "replace"
    providers: {
      "openai": {
        models: [
          { id: "gpt-4o", alias: "GPT-4o" },
          { id: "gpt-5.2-mini", alias: "GPT-5 Mini" }
        ]
      }
    }
  }
}
```

### 注意事项

- **合并是浅层**：同 provider 名会整体替换，不是深度合并模型列表
- **隐式 providers**：从 agent 目录发现的凭证文件也会参与合并
- **models.json 是最终产物**：配置变更需重启 Gateway 才生效

## 3️⃣ Canvas / Browser Control 进阶用法

### Canvas（节点 WebView）

Canvas 是节点设备（iOS/Android/macOS）上的 WebView，用于：
- 远程展示网页/仪表板
- A2UI (Agent-to-UI) 交互界面
- 截图监控

#### 核心命令

```bash
# 展示 URL 或本地文件
clawdbot nodes canvas present --node <id> --target https://example.com

# 带位置参数
clawdbot nodes canvas present --node <id> --target ./dashboard.html --x 0 --y 0 --width 800 --height 600

# 隐藏 Canvas
clawdbot nodes canvas hide --node <id>

# 导航
clawdbot nodes canvas navigate https://example.com --node <id>

# 执行 JS
clawdbot nodes canvas eval --node <id> --js "document.title"

# 截图（返回 MEDIA: 附件）
clawdbot nodes canvas snapshot --node <id> --format png --max-width 1200 --quality 0.9
```

#### A2UI (Agent-to-UI)

```bash
# 推送 JSONL payload（A2UI v0.8）
clawdbot nodes canvas a2ui push --node <id> --jsonl ./payload.jsonl

# 推送文本
clawdbot nodes canvas a2ui push --node <id> --text "Hello"

# 重置 A2UI 界面
clawdbot nodes canvas a2ui reset --node <id>
```

**注意**: 仅支持 A2UI v0.8，v0.9/createSurface 被拒绝。

### Browser Control（Chrome/Playwright）

Browser Control 是 Gateway 控制的浏览器自动化系统。

#### Profile 管理

| Profile | 用途 | 特点 |
|---------|------|------|
| `"clawd"` | 默认隔离浏览器 | Clawdbot 独立管理 |
| `"chrome"` | Chrome Relay 扩展 | 复用用户现有 Chrome 标签页 |
| `"custom"` | 自定义 CDP | 连接外部浏览器 |

#### Chrome Relay（推荐）

1. 安装 OpenClaw Browser Relay Chrome 扩展
2. 在目标标签页点击扩展图标（Badge ON = 已连接）
3. Agent 使用 `profile="chrome"` 控制该标签页

```javascript
// Agent 调用示例
browser({ action: "snapshot", profile: "chrome" });
browser({ action: "act", request: { kind: "click", ref: "e12" }, profile: "chrome" });
```

#### Snapshot 格式

**数字 Ref（Playwright aria-ref）**:
- `e12` = 元素 ID
- 适合快速定位，稳定性依赖 DOM 结构

**Role Ref**:
- `role=button,name="Submit"`
- 更稳定，依赖语义而非 DOM 结构

```javascript
// 获取 aria refs
browser({ action: "snapshot", refs: "aria" });  // 返回 e12, e13 等

// 获取 role refs
browser({ action: "snapshot", refs: "role" });  // 返回 role+name 组合
```

#### CDP 直连

```bash
# 启动 Chrome with remote-debugging
chrome --remote-debugging-port=9222

# 配置 Gateway
browser: {
  enabled: true,
  controlUrl: "http://127.0.0.1:9222"
}
```

#### 远程浏览器控制

场景：Gateway 在服务器，Chrome 在本地机器。

```bash
# 本地启动浏览器控制服务器
openclaw-cn browser serve --bind 127.0.0.1 --port 18791 --token SECRET

# 通过 Tailscale Serve 发布
tailscale serve https / http://127.0.0.1:18791

# Gateway 配置
browser: {
  enabled: true,
  controlUrl: "https://<magicdns>/"
}
```

## 4️⃣ 节点配对机制（iOS/Android/macOS）

### 配对流程

节点通过 **WebSocket** 连接 Gateway，使用 **设备配对** 进行认证。

#### 1. 启动节点

**iOS/Android App**:
- 打开 Clawdbot App
- 输入 Gateway 地址（如 `192.168.1.10:18789`）
- App 连接 WS，发送设备身份

**Headless Node Host** (Linux/Windows):
```bash
clawdbot node run --host <gateway-host> --port 18789 --display-name "Build Node"
```

**macOS Node Mode**:
- macOS 菜栏 App 连接 Gateway WS，作为节点暴露 `system.run` 等命令

#### 2. Gateway 创建配对请求

节点连接后，Gateway 创建待审批的设备配对请求。

#### 3. 审批配对

```bash
# 查看待审批
clawdbot devices list
clawdbot nodes pending

# 审批
clawdbot devices approve <requestId>
clawdbot nodes approve <requestId>

# 拒绝
clawdbot devices reject <requestId>
```

#### 4. 命名节点

```bash
# 方法1：启动时指定（持久化在节点）
clawdbot node run --display-name "Living Room iPad"

# 方法2：Gateway 覆盖
clawdbot nodes rename --node <id> --name "iPad"
```

### 节点能力矩阵

| 能力 | iOS | Android | macOS Node | Headless |
|------|-----|---------|------------|----------|
| `canvas.*` | ✅ | ✅ | ✅ | ❌ |
| `camera.*` | ✅ | ✅ | ✅ | ❌ |
| `screen.record` | ✅ | ✅ | ✅ | ❌ |
| `location.get` | ✅ | ✅ | ✅ | ❌ |
| `system.run` | ❌ | ❌ | ✅ | ✅ |
| `sms.send` | ❌ | ✅ | ❌ | ❌ |

### 节点必须前台

`canvas.*` 和 `camera.*` 要求节点 App **前台运行**。
后台调用返回 `NODE_BACKGROUND_UNAVAILABLE`。

### Exec 绑定节点

```bash
# 全局默认
clawdbot config set tools.exec.node "iPad"

# Per-agent 覆盖
clawdbot config set agents.list[0].tools.exec.node "Build Node"

# 清除（允许任意节点）
clawdbot config unset tools.exec.node
```

### Exec Approvals（节点安全）

节点上的 `system.run` 受 `~/.openclaw/exec-approvals.json` 限制。

```bash
# 添加允许列表
clawdbot approvals allowlist add --node <id> "/usr/bin/git"
```

## 5️⃣ Webhooks + Tailscale 远程访问

### Webhooks 入口

Gateway 暴露 HTTP webhook 端点，允许外部系统触发 Agent。

#### 启用配置

```json5
{
  hooks: {
    enabled: true,
    token: "${OPENCLAW_HOOKS_TOKEN}",   // 必需
    path: "/hooks",                      // 默认
    allowedAgentIds: ["hooks", "main"],  // 限制 agentId
    defaultSessionKey: "hook:ingress",   // 默认会话
    allowRequestSessionKey: false        // 禁止请求覆盖 sessionKey
  }
}
```

#### 认证方式

```bash
# 推荐：Bearer Token
curl -H 'Authorization: Bearer SECRET'

# 备用：x-openclaw-token
curl -H 'x-openclaw-token: SECRET'

# 查询字符串 token 被拒绝（返回 400）
```

#### 端点

| 端点 | 功能 | 返回 |
|------|------|------|
| `POST /hooks/wake` | 触发心跳/系统事件 | `200` |
| `POST /hooks/agent` | 运行独立 Agent 回合 | `202` |
| `POST /hooks/<name>` | 自定义映射 | `200/202` |

#### Wake 示例

```bash
curl -X POST http://127.0.0.1:18789/hooks/wake \
  -H 'Authorization: Bearer SECRET' \
  -H 'Content-Type: application/json' \
  -d '{"text":"收到新邮件","mode":"now"}'
```

#### Agent 示例

```bash
curl -X POST http://127.0.0.1:18789/hooks/agent \
  -H 'Authorization: Bearer SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "message": "总结收件箱",
    "name": "邮件",
    "model": "openai/gpt-5.2-mini",
    "deliver": true,
    "channel": "telegram",
    "to": "+15551234567"
  }'
```

#### 自定义映射

```json5
{
  hooks: {
    mappings: {
      gmail: {
        match: { source: "gmail" },
        action: "agent",
        message: { template: "处理 Gmail: {{subject}}" }
      }
    }
  }
}
```

### Tailscale 远程访问

#### 模式

| 模式 | 用途 | 认证 |
|------|------|------|
| `"serve"` | 仅尾网访问 | 支持 Tailscale 身份头 |
| `"funnel"` | 公共 HTTPS | 必须使用密码 |
| `"off"` | 无 Tailscale 自动化 | - |

#### Serve 配置（尾网）

```json5
{
  gateway: {
    bind: "loopback",
    tailscale: { mode: "serve" },
    auth: { allowTailscale: true }
  }
}
```

访问：`https://<magicdns>/`

**身份验证**: Tailscale Serve 注入身份头（`tailscale-user-login`），Gateway 通过 `tailscale whois` 验证。

#### Funnel 配置（公共）

```json5
{
  gateway: {
    bind: "loopback",
    tailscale: { mode: "funnel" },
    auth: { mode: "password", password: "${OPENCLAW_GATEWAY_PASSWORD}" }
  }
}
```

**安全**: Funnel 必须使用密码认证，避免公开暴露无保护。

#### Tailnet 直接绑定

```json5
{
  gateway: {
    bind: "tailnet",  // 直接监听 Tailnet IP
    auth: { mode: "token" }
  }
}
```

访问：`http://<tailscale-ip>:18789`

**注意**: 回环（`127.0.0.1`）在此模式不工作。

#### 远程浏览器控制 + Tailscale

```bash
# 本地机器运行 Chrome + 浏览器控制服务器
openclaw-cn browser serve --bind 127.0.0.1 --port 18791 --token SECRET
tailscale serve https / http://127.0.0.1:18791

# Gateway（远程服务器）配置
browser: {
  enabled: true,
  controlUrl: "https://<magicdns>/"
}
export OPENCLAW_BROWSER_CONTROL_TOKEN="SECRET"
```

### 安全最佳实践

1. **Webhooks**: 使用专用 token，不重用 Gateway token
2. **Tailscale**: Serve 优于 Funnel（尾网更安全）
3. **Funnel**: 必须设置密码，限制 `allowedAgentIds`
4. **Session Key**: 禁止 `allowRequestSessionKey`，或限制前缀
5. **网络**: 保持 webhook 在回环/尾网/受信任反向代理后

## 💡 关键要点总结

1. **nativeSkills "auto"** = Discord/Telegram 默认启用，Slack 需手动
2. **models.mode "merge"** = 浅合并 providers，新覆盖旧同名键
3. **Canvas** = 节点 WebView，需前台运行
4. **Browser Control** = Chrome Relay 或 CDP，支持远程
5. **节点配对** = WS 连接 + devices approve
6. **Webhooks** = HTTP 入口，token 认证，支持 agent 触发
7. **Tailscale Serve** = 尾网 HTTPS + 身份验证，比 Funnel 更安全
