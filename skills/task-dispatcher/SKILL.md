# Task Dispatcher - 任务分发技能

## 用途
将用户的聊天请求解析为具体任务，分配给子代理或 Cron 任务处理。

## 触发条件
用户在主会话中提出需要多步骤处理或专业领域的需求时。

## 任务类型识别

### 📊 数据分析类
**关键词**: 统计、分析、报告、数据、使用情况
**分配**: 使用 `qwen3.5-plus` 模型生成报告

### 💻 代码开发类
**关键词**: 代码、编程、功能、开发、测试、bug
**分配**: 使用 `qwen3-coder-plus` 或 `qwen3-coder-next`

### 🔧 系统运维类
**关键词**: 监控、健康、状态、检查、日志、备份
**分配**: 使用 `qwen3-coder-plus`

### 📅 日程提醒类
**关键词**: 提醒、日程、会议、计划、待办
**分配**: 使用 `minimax-m2.5` 或创建一次性 Cron 任务

### 📰 信息收集类
**关键词**: 新闻、搜索、查询、天气、邮件
**分配**: 使用 `kimi-k2.5` 或 `qwen3.5-plus`

### 🎯 综合规划类
**关键词**: 规划、计划、总结、回顾、建议
**分配**: 使用 `glm-5` 或 `qwen3.5-plus`

## 使用方式

### 方式 1: 子代理 (实时)
```bash
# 用户说："帮我分析一下这个项目的代码结构"
# 主会话调用 sessions_spawn 创建子代理
```

### 方式 2: Cron 任务 (定时)
```bash
# 用户说："每天早上检查代码质量"
# 主会话调用 cron.add 创建定时任务
```

### 方式 3: 直接处理
```bash
# 简单请求直接由主会话处理
```

## 示例

**用户**: "帮我规划一个 Python Web 项目，需要用户登录和数据展示功能"

**处理流程**:
1. 识别为"代码开发类" + "综合规划类"
2. 调用子代理 (`qwen3-coder-plus`) 生成技术规划
3. 创建项目待办清单
4. 建议创建每日进度检查 Cron 任务

**用户**: "统计一下这周各模型的使用情况"

**处理流程**:
1. 识别为"数据分析类"
2. 读取 sessions.json 和 cron 任务记录
3. 生成使用报告
4. 建议优化方案

## 注意事项

1. **实时性要求高** → 使用子代理 (sessions_spawn)
2. **周期性任务** → 使用 Cron 任务 (cron.add)
3. **简单查询** → 主会话直接处理
4. **跨任务协作** → 由"任务协调员"Cron 整合

## 模型选择速查

| 任务类型 | 推荐模型 |
|---------|---------|
| 通用对话 | qwen3.5-plus |
| 代码开发 | qwen3-coder-plus / qwen3-coder-next |
| 长文档/报告 | glm-5 |
| 多模态/邮件 | kimi-k2.5 |
| 日程/办公 | minimax-m2.5 |
