## 一、模型分工（专用于 Obsidian）

### 1. 知识节点生成与整理（主模型）

**模型**: `qwen3.5-plus`

**职责**:
- 创建 Obsidian 笔记
- 扩展知识内容
- 建立 [[双链]]
- 组织知识结构

**输出要求**:
- [ ] 结构化输出（标题 + 分段 + 关联）
- [ ] 至少建立 2 个以上关联节点
- [ ] 包含概述、要点、说明
- [ ] 元数据完整（日期、标签）

**适用场景**:
- 创建新知识笔记
- 扩展已有知识节点
- 建立双链关联
- 整理知识结构

---

### 2. 复盘与总结（分析模型）

**模型**: `glm-5`

**职责**:
- 生成复盘笔记（Issues / Decisions）
- 提供原因分析
- 输出优化建议

**输出要求**:
- [ ] 必须包含"原因 + 改进"
- [ ] 不允许只描述现象
- [ ] 有数据或事实支撑
- [ ] 建议可执行

**适用场景**:
- 项目复盘总结
- 问题分析记录
- 决策过程回顾
- 经验教训整理

---

### 3. 结构优化与格式控制（结构模型）

**模型**: `qwen3-coder-plus`

**职责**:
- 修正 Markdown 格式
- 生成 Mermaid 图
- 清理混乱笔记结构
- 统一 YAML 头信息

**输出要求**:
- [ ] 不改变内容语义
- [ ] 只优化结构与格式
- [ ] 语法正确（Markdown/Mermaid）
- [ ] 格式规范统一

**适用场景**:
- Markdown 格式修正
- Mermaid 图表生成
- YAML 头信息统一
- 笔记结构整理

## 二、自动调度逻辑

### 任务类型识别

根据任务描述中的关键词自动选择模型：

| 关键词 | 模型 | 任务类型 |
|--------|------|---------|
| 笔记 / Obsidian / 知识 / 节点 / 双链 / 图谱 | qwen3.5-plus | 知识节点 |
| 复盘 / 总结 / 分析 / 原因 / 决策 / 经验 | glm-5 | 复盘分析 |
| 结构 / 格式 / markdown / 图 / Mermaid / YAML | qwen3-coder-plus | 结构优化 |

### 优先级规则

1. **精确匹配优先**: "Obsidian"、"Mermaid"、"YAML" → 直接对应模型
2. **语义匹配次之**: "笔记"、"复盘"、"格式" → 对应模型
3. **模糊匹配最后**: "整理"、"优化" → 根据上下文判断

## 三、失败与降级机制

### 切换策略

```
第 1 次失败 → 重试当前模型
第 2 次失败 → 切换模型
    ├─ qwen3.5-plus → glm-5
    ├─ glm-5 → qwen3.5-plus
    └─ 其他 → qwen3-coder-plus
第 3 次失败 → 使用 qwen3-coder-plus 兜底
第 4 次失败 → 记录错误，发送告警
```

### 失败原因分类

**模型错误**:
- API 限制（请求超限）
- 认证失败（Token 过期）
- 超时错误（响应超时）

**质量错误**:
- 输出不符合要求
- 缺少必需内容（如双链、原因分析）
- 格式错误

**处理策略**:
- 模型错误 → 切换模型重试
- 质量错误 → 重新生成（相同模型）

## 四、知识库保护机制

### 1. 重复检测

**规则**: 不允许创建重复主题文件

**检测方法**:
1. 检查 `knowledge/` 目录下是否存在相同标题
2. 检查是否存在相似主题（关键词匹配）
3. 检查双链是否已存在

**处理方式**:
- 发现重复 → 提示用户，建议更新现有笔记
- 发现相似 → 建议合并或建立关联

---

### 2. 相似笔记合并

**规则**: 相似笔记必须合并

**合并条件**:
- 主题相似度 > 80%
- 内容重叠 > 50%
- 创建时间相近（< 7 天）

**合并策略**:
1. 保留较完整的笔记
2. 将另一篇内容整合进去
3. 建立重定向双链
4. 删除或归档重复笔记

---

### 3. 孤立笔记处理

**规则**: 孤立笔记必须补充关联

**孤立定义**:
- 无任何双链关联
- 未被其他笔记引用
- 创建时间 > 7 天

**处理方式**:
1. 分析笔记内容，识别潜在关联
2. 建议至少 2 个关联节点
3. 自动创建缺失的空壳笔记
4. 建立双向链接

---

### 4. Inbox 工作流

**规则**: 所有新笔记先进入 `00-Inbox/`，再由整理任务归档

**流程**:
```
创建笔记
    ↓
写入 00-Inbox/笔记标题.md
    ↓
知识整理员扫描（每天 02:00）
    ↓
识别主题和分类
    ↓
移动到正式目录（knowledge/知识/、knowledge/项目/等）
    ↓
建立双链关联
    ↓
生成整理报告
```

**目录结构**:
```
E:\software\Obsidian\vault\
├── 00-Inbox/           # 新笔记缓冲区
├── knowledge/
│   ├── 知识/           # 通用概念、理论
│   ├── 项目/           # 进行中任务
│   ├── 问题/           # 问题及解决方案
│   └── 系统设计/       # 架构、规范
└── ...
```

## 六、使用方式

### 方式 1: 手动指定模型

```powershell
# 知识节点创建
openclaw sessions spawn --model "dashscope-coding-plan/qwen3.5-plus" --task "创建关于 AI 的笔记"
obsidian-cli create "知识/AI 基础" --content "..."

# 复盘总结
openclaw sessions spawn --model "dashscope-coding-plan/glm-5" --task "分析本周项目失败原因"
obsidian-cli create "问题/项目失败分析" --content "..."

# 结构优化
openclaw sessions spawn --model "dashscope-coding-plan/qwen3-coder-plus" --task "整理 Markdown 格式"
obsidian-cli edit "知识/AI 基础" --append "..."
```

### 方式 2: 使用调度器脚本

```powershell
# 分析任务类型
.\obsidian-model-scheduler.ps1 -AnalyzeOnly -Task "创建 Obsidian 知识笔记"
# 输出：qwen3.5-plus

# 执行任务（带失败重试）
.\obsidian-model-scheduler.ps1 -Task "创建笔记" -VaultPath "E:\software\Obsidian\vault"
```
