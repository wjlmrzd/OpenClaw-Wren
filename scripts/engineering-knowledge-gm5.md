- 独立管理：使用 glm-5 模型进行专业化管理
- 主题聚焦：专注于工程领域，不与其他篇章重叠

## 三、glm-5 模型管理策略

### 适用任务

| 任务类型 | 说明 | 模型 |
|---------|------|------|
| 工程技术笔记创建 | CAD、脚本、工具等技术文档 | glm-5 |
| 工程复盘分析 | 项目复盘、问题分析、原因追溯 | glm-5 |
| 经验总结 | 工程经验、技术踩坑记录 | glm-5 |
| 流程优化 | 工艺流程优化建议 | glm-5 |
| 架构设计 | 系统架构、技术方案 | glm-5 |

### glm-5 核心优势

```
glm-5 特点:
├─ 强推理能力 → 复杂工程问题分析
├─ 代码理解 → CAD 脚本、自动化工具
├─ 原因追溯 → 故障分析、问题复盘
└─ 结构化输出 → 工程文档规范化
```

## 四、知识互通机制

### 与主会话互通

1. **直接引用**: 主会话可直接引用工程知识篇章中的笔记
2. **双链建立**: 使用 [[工程知识/文件名]] 建立跨篇章链接
3. **知识同步**: 重要工程知识同步到 MEMORY.md

### 关键词触发

当主会话出现以下关键词时，优先检索工程知识篇章：
- CAD、AutoCAD、建模
- 脚本、自动化、工具
- 架构、设计、系统
- 工艺、流程、优化
- 工程、项目、实践

## 五、使用方式

### 主会话触发

```
用户: "我在做 CAD 属性块转换，遇到了 XXX 问题"
   ↓
识别: CAD 相关 → 检索 工程知识/CAD与建模/
   ↓
回答: 优先使用 工程知识篇章 中的相关笔记
   ↓
补充: 如有必要，创建新的工程知识笔记
```

### 独立管理

```
用户: "帮我记录一个 CAD 技巧到工程知识篇章"
   ↓
模型: glm-5
   ↓
输出: 标准化的工程知识笔记格式
   ↓
存储: knowledge/工程知识/CAD与建模/
```
