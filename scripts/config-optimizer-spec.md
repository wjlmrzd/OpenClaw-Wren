# Config Optimizer 规格文档

## 任务目标

每天定点检查并优化 OpenClaw 配置（`openclaw.json`），确保配置始终处于最佳状态。

## 优化检查项

### 1. 模型能力核实
- 检查各模型的 `input` 字段是否完整（text-only 模型应无 image，多模态应有 image）
- 检查 `reasoning` 字段是否与模型能力匹配
- 检查 `maxTokens` 是否在合理范围

### 2. 配置一致性
- `agents.defaults.models` 中的别名映射是否与 providers 中的模型 ID 一致
- channel 配置中的 proxy 是否可达
- 环境变量引用是否都能正确解析

### 3. 日志级别优化
- 生产环境应使用 `info` 而非 `debug`（减少日志噪音和磁盘占用）
- 检查是否存在未使用的配置字段

### 4. 模型配置优化
- 确认所有已安装 Coding Plan 模型都在配置中
- 检查是否有重复或无效的模型条目

### 5. 安全检查
- 检查是否有硬编码的敏感信息（而非环境变量引用）
- 检查 `allowInsecureAuth` 等危险设置

### 6. 版本信息更新
- 更新 `meta.lastTouchedAt` 为当前时间
- 记录本次优化变更内容

## 优化策略

| 优化项 | 风险等级 | 操作方式 |
|--------|---------|---------|
| 日志级别 debug→info | 🟡 中 | 自动优化 |
| input 字段修正 | 🟡 中 | 仅报告，不自动修改 |
| reasoning 字段 | 🟡 中 | 仅报告，不自动修改 |
| maxTokens 调优 | 🟢 低 | 自动优化（保守调整） |
| meta.lastTouchedAt | 🟢 低 | 自动更新 |
| 配置一致性检查 | 🟢 低 | 仅报告 |

## 输出报告

每次执行后生成报告，包含：
- 📋 检查项清单及结果
- 🔧 执行的自动优化
- ⚠️ 需要人工审核的项目
- 📊 与上次配置对比（如有变更）

## 执行频率

每天 **03:30**（在每日维护员之后运行）
