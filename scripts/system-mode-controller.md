# OpenClaw 运行模式控制器

## 三种运行模式

### 1. 🟢 正常模式 (Normal)
**默认状态** - 所有 Agent 按原计划运行

### 2. 🟡 降载模式 (Reduced Load)
**触发条件** (满足任意一个):
- API 使用超过 80%
- 内存或 CPU 超过 85%
- 连续失败任务 ≥ 3

**执行策略**:
- 降低非关键任务频率 (50%)
- 暂停低优先级任务:
  - ❌ 每日早报
  - ❌ 网站监控
  - ❌ 运动提醒
  - ❌ 周报
- 保留核心任务:
  - ✅ 健康监控员
  - ✅ 故障自愈员
  - ✅ 事件协调员

### 3. 🔴 安全模式 (Safe Mode)
**触发条件** (满足任意一个):
- 内存 > 95%
- 磁盘 > 95%
- Gateway 无法正常运行
- 关键任务连续失败 ≥ 5 次

**执行策略**:
- 停止所有非必要任务
- 仅保留:
  - ✅ 故障自愈员
  - ✅ 健康监控员
  - ✅ 事件协调员
- 强制执行日志清理
- 发送 Telegram 紧急通知

## 资源阈值

### 内存
| 使用率 | 动作 |
|--------|------|
| > 85% | 触发清理，进入降载模式 |
| > 90% | 准备重启 Gateway |
| > 95% | 强制进入安全模式 |

### 磁盘
| 使用率 | 动作 |
|--------|------|
| > 85% | 自动清理日志 |
| > 90% | 压缩历史文件 |
| > 95% | 删除非关键数据 |

### API 配额
| 使用率 | 动作 |
|--------|------|
| > 70% | 记录趋势 |
| > 80% | 降低任务频率 |
| > 90% | 停止低优先级任务 |

## 模式切换流程

```
正常模式
  ↓ 检测到资源压力
降载模式 (暂停低优先级任务)
  ↓ 资源恢复
正常模式

降载模式
  ↓ 资源继续恶化
安全模式 (仅保留核心任务)
  ↓ 资源恢复
降载模式 → 正常模式
```

---

## 紧急通知条件

安全模式下立即发送 Telegram:
- 内存 > 95%
- 磁盘 > 95%
- Gateway 重启失败 ≥ 3 次
- 核心任务连续失败 ≥ 5 次
