# 🔕 情境感知静默 (Context-Aware Silence)

**职责：** 根据时间、场景、用户状态智能调整通知策略

---

## 核心功能

### 1. 时间感知

#### 时段划分

| 时段 | 时间 | 通知策略 |
|------|------|---------|
| 🌅 早晨 | 06:00-09:00 | 正常通知（可发送早报/提醒） |
| ☀️ 工作时间 | 09:00-12:00, 14:00-18:00 | 正常通知（所有级别） |
| 🌆 傍晚 | 18:00-22:00 | 正常通知（可发送总结） |
| 🌙 睡眠时间 | 22:00-06:00 | **静默**（仅紧急） |

#### 静默时段规则

**22:00-06:00 期间：**
- 🟢 信息类 → 累积到 06:00 统一发送
- 🟡 注意类 → 累积到 06:00 统一发送
- ⚠️ 警告类 → 仅当连续≥3 次时发送
- 🔴 紧急类 → **立即发送**（系统故障）

**例外情况（可发送）：**
- 用户主动询问
- 系统故障无法自愈（Gateway 宕机、连续失败≥5 次）
- 资源危机（内存>95%、磁盘>95%）

### 2. 场景感知

#### 场景检测

| 场景 | 检测方法 | 通知策略 |
|------|---------|---------|
| 会议中 | 日历事件包含"会议" | 仅紧急 |
| 外出 | 日历事件包含"外出"/"出差" | 仅紧急 |
| 专注时间 | 日历事件包含"专注"/"勿扰" | 完全静默 |
| 周末 | 星期六/日 | 降低频率（50%） |
| 节假日 | 日历标记/国家节假日 API | 降低频率（80%） |

#### 动态调整

```yaml
场景：会议中
调整:
  - 非紧急通知 → 延迟到会议结束
  - 紧急通知 → 立即发送
  - 摘要频率 → 每小时一次 → 每天一次

场景：专注时间
调整:
  - 所有通知 → 完全静默
  - 累积日志 → 记录到 memory/silenced-events.md
  - 恢复时间 → 专注结束后统一发送
```

### 3. 用户状态感知

#### 在线状态检测

| 状态 | 检测方法 | 通知策略 |
|------|---------|---------|
| 在线 | Telegram 最后在线 < 5 分钟 | 正常通知 |
| 忙碌 | Telegram 状态="busy" | 仅重要通知 |
| 离线 | Telegram 最后在线 > 1 小时 | 累积发送 |
| 睡眠 | 时间 + 日历 + 历史行为 | 仅紧急 |

#### 响应行为学习

```
用户行为模式学习：
- 22:00 后不回复消息 → 推断为睡眠时间
- 会议期间不回复 → 推断为忙碌
- 周末回复频率低 → 推断为休息时间

自适应调整：
- 学习时间窗口（如 23:00-07:00）→ 自动设为静默
- 学习忙碌场景 → 自动降低通知频率
```

### 4. 通知优先级

#### 优先级定义

| 级别 | 代码 | 描述 | 示例 |
|------|------|------|------|
| 🟢 | INFO | 信息类 | 任务完成、日常报告 |
| 🟡 | NOTICE | 注意类 | 单次失败、资源警告 |
| ⚠️ | WARNING | 警告类 | 连续失败≥3 次、资源紧张 |
| 🔴 | CRITICAL | 紧急类 | 系统故障、资源危机 |
| 🚨 | EMERGENCY | 灾难类 | Gateway 宕机、数据丢失风险 |

#### 静默策略矩阵

| 优先级 | 工作时间 | 睡眠时间 | 会议中 | 专注时间 |
|--------|---------|---------|--------|---------|
| 🟢 INFO | ✅ | ❌ (累积) | ❌ | ❌ |
| 🟡 NOTICE | ✅ | ❌ (累积) | ❌ | ❌ |
| ⚠️ WARNING | ✅ | ⚠️ (≥3 次) | ❌ | ❌ |
| 🔴 CRITICAL | ✅ | ✅ | ✅ | ✅ |
| 🚨 EMERGENCY | ✅ | ✅ | ✅ | ✅ |

## 实现方式

### 方式 1: Agent 模式（推荐）

**创建 Agent:** 🔕 静默管理员

**职责：**
- 每 30 分钟检查时间/场景/状态
- 更新 silence-state.json
- 提供 API 供其他 Agent 查询
- 累积和分发静默期间的通知

**执行频率：** 每 30 分钟

### 方式 2: 工具函数模式

**创建工具：** silence-tools.ps1

**函数：**
- `Get-CurrentContext()` - 获取当前时间/场景/状态
- `ShouldNotify($priority)` - 判断是否应该发送通知
- `AccumulateEvent($event)` - 累积静默期间的事件
- `FlushAccumulatedEvents()` - 静默结束后发送累积事件

### 方式 3: 配置模式（最简单）

**修改：** 各 Agent 的通知逻辑

**规则：**
```powershell
# 在每个 Agent 的通知前添加检查
if (Test-IsQuietHours) {
    if ($priority -ne "critical" -and $priority -ne "emergency") {
        # 累积到非静默时段
        Add-ToAccumulatedEvents
        return
    }
}
```

## 与其他 Agent 的联动

### 与 Event Hub 联动

```
Event Hub 检测到事件
    ↓
查询静默状态
    ↓
静默中 → 累积事件
非静默 → 立即发送
```

### 与日历检查联动

```
日历检查员检测到会议
    ↓
更新 silence-state.json
    ↓
Event Hub 调整通知策略
```
